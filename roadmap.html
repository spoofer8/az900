§ 01 · Strategy

How the fourteen days break.

The plan follows the three official skill domains, then adds a fourth phase for mastery. Study time compounds: concepts on day one anchor everything you learn in week two. Days are sized to fit around a job — roughly two to three hours of focused work each, front-loaded on weekdays with lighter sessions on weekends.

2 h reading
30 m setup

Cloud computing, shared responsibility, and CapEx vs OpEx

Ground zero. What "the cloud" actually is, why organisations move to it, and the single model that governs who owns what between the customer and the cloud provider. Also: create your free Azure account today so you can practise from Day 4 onward.

Notes 1.1 Cloud computing Quiz 1.1 Cloud computing

Learn

  • Definition of cloud computing and its role in modern IT
  • The shared responsibility model — where the customer's line ends and the provider's begins in IaaS / PaaS / SaaS
  • CapEx (buy hardware) vs OpEx (rent capacity) — the finance lens that drives most cloud decisions
  • The consumption-based pricing model — pay only for what you use

Study

  • MS Learn · Describe cloud computing
  • MS Learn · Describe cloud service types (skim; deep tomorrow)
  • Azure docs · Shared responsibility in the cloud
  • Create a free Azure account · $200 credit + 12 months of services

Hands-on

Sign up for the free Azure account. Sign in to the Azure portal and pin these blades to your dashboard: Subscriptions, Resource groups, Cost Management, Microsoft Entra ID.

Open Cloud Shell, choose Bash, and run az account show to confirm your subscription is active.

Self-check

  • In IaaS, who patches the guest operating system — customer or provider?
  • Which pricing model shifts IT spend from CapEx to OpEx and why does that matter to a CFO?
  • Name one workload where the consumption model saves money and one where it doesn't.

Benefits, cloud models, and service categories

The vocabulary the exam expects you to use fluently: availability vs reliability vs scalability; public vs private vs hybrid vs multi-cloud; IaaS vs PaaS vs SaaS vs serverless. Distractor questions live in these boundaries.

Notes 1.2 Cloud benefits Notes 1.3 Service types Quiz 1.2 Cloud benefits Quiz 1.3 Service types

Learn

  • Benefits: high availability, scalability (vertical vs horizontal), elasticity, reliability, predictability, security, governance, manageability
  • Cloud models: public (rented from a hyperscaler), private (single tenant), hybrid (both), multi-cloud (more than one provider) — and their trade-offs
  • Service categories: IaaS (VMs, networks), PaaS (App Service, SQL DB), SaaS (Microsoft 365) — with the shared-responsibility line for each
  • Serverless as a consumption pattern (Functions, Logic Apps) — event-driven, pay-per-execution, no server management

Study

  • MS Learn · Describe the benefits of using cloud services
  • MS Learn · Cloud service types (full read today)
  • Azure · What is cloud computing?
  • John Savill's Technical Training · YouTube — search "AZ-900 Study Cram v3" and watch the concepts section

Hands-on

In the portal, browse the Marketplace. Filter by category and identify at least one example of each service type: an IaaS offering (a Linux VM image), a PaaS offering (App Service or Azure SQL Database), and a SaaS offering (a third-party image like a WordPress-managed stack).

Do not create anything yet — this is a taxonomy exercise.

Self-check

  • Scalability vs elasticity — define both without using either word.
  • You're moving a legacy on-prem SQL server to Azure but keeping some workloads in your datacentre. Which cloud model?
  • Which service category has the customer responsible for the guest OS but not the physical host?
  • Is Azure Functions serverless, PaaS, both, or neither?

Phase II · Architecture
2 h reading
45 m portal lab

Azure's core architectural components

Regions, Availability Zones, and the four-level resource hierarchy. This is the single most heavily tested topic in Domain 2 — get it precise. Know the difference between an availability zone and an availability set, and where region pairs fit in disaster recovery.

Notes 2.1 Core architecture Quiz 2.1 Core architecture

Learn

  • Regions: 60+ globally, each with one or more datacentres. Some are sovereign (Azure Government, Azure China 21Vianet)
  • Region pairs: replicated for DR, usually 300+ miles apart, updates staggered
  • Availability Zones: minimum three physically separate zones in a zone-enabled region — used for high-availability across a region
  • Hierarchy: Management groups → subscriptions → resource groups → resources
  • Azure Resource Manager (ARM) as the deployment and management layer for every Azure resource

Study

  • MS Learn · Core architectural components of Azure
  • Azure docs · Availability zones overview
  • Azure docs · Azure Resource Manager overview
  • Azure global infrastructure map

Hands-on

Create a resource group in your subscription — pick a region close to you. Note the exact URL of the resource group; it will contain your subscription ID and resource-group name, showing the hierarchy in action.

In Cloud Shell run az group list --output table and az account show --output table to see the same hierarchy from the CLI.

Self-check

  • Availability zone vs availability set — which protects against a datacentre failure and which against a rack failure?
  • What is a region pair and how many miles typically separate the two regions?
  • Can a resource group span multiple subscriptions? Can a resource belong to two resource groups?
  • What is the top of the Azure resource hierarchy?

Phase II · Compute
2 h reading
1 h lab

Compute services — VMs, containers, apps, and serverless

Six flavours of compute, each with a distinct sweet spot. Know which one to pick given a scenario: lift-and-shift, web app, microservices, batch, event-driven, or virtual desktop. This is where scenario questions concentrate.

Notes 2.2 Compute & networking Quiz 2.2 Compute & networking

Learn

  • Azure Virtual Machines — full IaaS control. VM Scale Sets for horizontal scale; Availability Sets for rack-level HA
  • Azure App Service — PaaS for web apps, APIs, mobile back-ends. Multi-language, auto-scale, deployment slots
  • Azure Container Instances vs Container Apps vs Azure Kubernetes Service (AKS)
  • Azure Functions — serverless code execution, triggered by events, priced per execution
  • Azure Virtual Desktop — desktop-as-a-service, multi-session Windows

Study

  • MS Learn · Describe Azure compute and networking services
  • Azure docs · Compute decision tree
  • Azure Functions overview
  • AKS overview

Hands-on

Create a small Linux VM (B1s size, in the resource group from yesterday). Note the resources auto-created alongside it: virtual network, subnet, NIC, public IP, disk, network security group. This is the "unit of compute" you'll deconstruct on Day 6 (networking) and Day 7 (storage).

Important: stop and deallocate the VM the moment you finish so it stops billing.

Self-check

  • You have a small event-driven Python script that runs a few times an hour. VM or Function?
  • You have a monolithic Windows app you need to move to Azure without code changes. Which service?
  • Container Instances vs AKS — pick the right one for a one-off batch job and for a production microservices platform.
  • What is the SLA difference between a single VM, a VM in an availability set, and a VM across availability zones?

Phase II · Networking
2 h reading
45 m lab

Networking — VNets, gateways, and load balancing

Azure networking is a rich topic; AZ-900 stays fundamentals. Focus on the identity of each service: what problem it solves, how it differs from adjacent services. Load-balancer questions are notoriously fond of testing OSI-layer distinctions.

Notes 2.2 Networking half Quiz 2.2 Compute & networking

Learn

  • Virtual Networks (VNets) and subnets — isolated IP space in Azure
  • VNet peering — connect VNets privately (local and global)
  • VPN Gateway (site-to-site & point-to-site over public internet) vs ExpressRoute (private connection through a partner, no public internet)
  • Azure DNS and Private DNS zones
  • Load balancing: Load Balancer (L4), Application Gateway (L7, in-region), Front Door (L7, global), Traffic Manager (DNS-based)

Study

  • Azure docs · Virtual Network overview
  • Azure docs · ExpressRoute introduction
  • Azure docs · Load-balancing decision tree
  • Azure docs · Azure Front Door

Hands-on

Open yesterday's VM's associated virtual network. Explore subnets, address space, and the network security group attached to the NIC. Try adding an inbound rule to allow port 80.

Then delete the rule to leave the VM secure. Do not expose a live VM to the internet unnecessarily.

Self-check

  • You need a dedicated private circuit from your datacentre to Azure with 10 Gbps and no public internet transit. Which service?
  • Application Gateway vs Front Door — one is regional, one is global. Which is which?
  • What OSI layer does Azure Load Balancer operate at, and what does that let it do (or not do)?
  • Two VNets in different regions — what connects them privately?

Phase II · Storage
2 h reading
45 m lab

Storage services, redundancy, and access tiers

Two acronym clusters you must fully own: redundancy (LRS, ZRS, GRS, GZRS + read-access variants) and tiers (Hot, Cool, Cold, Archive). Expect at least two exam questions on each. Also cover migration tools — Azure Migrate, Data Box, AzCopy, File Sync.

Notes 2.3 Storage Quiz 2.3 Storage

Learn

  • Storage account service types: Blob, File, Queue, Table, Disk
  • Redundancy: LRS (3 copies, one datacentre), ZRS (3 zones, one region), GRS (6 copies across two regions), GZRS (ZRS + geo), plus RA-GRS / RA-GZRS for read access to the secondary
  • Access tiers for Blob: Hot > Cool > Cold > Archive. Storage cost falls each step, access cost and latency rise (Archive requires rehydration, up to 15 hours)
  • Migration & movement: Azure Migrate (discover / assess / migrate), Data Box (physical device for offline transfer), AzCopy (CLI), Storage Explorer (desktop GUI), Azure File Sync (hybrid SMB shares)

Study

  • MS Learn · Describe Azure storage services
  • Azure docs · Storage redundancy
  • Azure docs · Blob access tiers
  • Azure docs · Azure Migrate

Hands-on

Create a storage account. During creation, walk through every option — redundancy, performance, access tier — and note how the estimated cost changes. Create a Blob container, upload a file, change its tier to Cool, then Archive. Try to download the archived blob and observe the rehydration behaviour.

Self-check

  • You need protection against a full-region outage and read access to the secondary region. Which redundancy?
  • Data accessed rarely, if ever, but that must be retained for seven years. Which access tier?
  • You're shipping 100 TB of on-prem data to Azure over a slow uplink. What service saves you weeks?
  • ZRS protects against what specific failure that LRS does not?

Phase II · Identity & Security
2.5 h reading
30 m lab

Identity, access, and security — Entra ID and Zero Trust

Microsoft has renamed Azure AD to Microsoft Entra ID; the exam uses the new name. Nail authentication vs authorisation, MFA vs Conditional Access, and the three principles of Zero Trust. Also: know Defender for Cloud, Sentinel, Key Vault, and Azure Firewall by their role.

Notes 2.4 Identity, access & security Quiz 2.4 Identity, access & security

Learn

  • Microsoft Entra ID — identity and access management for cloud apps
  • Authentication (who you are) vs Authorisation (what you can do)
  • MFA (something you know / have / are), Conditional Access (rules that evaluate signals like device, location, risk), SSO, passwordless (FIDO2, Microsoft Authenticator, Windows Hello)
  • External identities — B2B collaboration, B2C for customer apps
  • Zero Trust — verify explicitly, use least-privilege access, assume breach
  • Defense in depth — layered controls: data, application, compute, network, perimeter, identity, physical
  • Security tooling: Microsoft Defender for Cloud (CSPM + workload protection, Secure Score), Microsoft Sentinel (SIEM/SOAR), Azure Key Vault, Azure Firewall, Azure DDoS Protection

Study

  • MS Learn · Describe Azure identity, access, and security
  • Microsoft Entra ID · What is it?
  • Zero Trust overview
  • Microsoft Defender for Cloud

Hands-on

In the Entra ID blade, create a new user (a test identity, not your real one). Explore the Security section — Conditional Access, MFA, Identity Protection. Do not apply Conditional Access policies to your admin account.

Open Microsoft Defender for Cloud and look at your Secure Score for the subscription. Read one recommendation and note the remediation steps — don't apply them yet.

Self-check

  • Authentication or authorisation: an app checks that you're a member of the "billing-admin" group before showing invoices?
  • List the three principles of Zero Trust from memory.
  • Sentinel vs Defender for Cloud — which is the SIEM and which is the CSPM?
  • Which service securely stores secrets, keys, and certificates so applications don't hard-code them?

Phase III · Cost
1.5 h reading
1 h lab

Cost management — calculators, billing, and savings

Six factors move the Azure bill; two calculators help you plan; one blade helps you police it. Own the difference between the Pricing Calculator (planning a new workload) and the TCO Calculator (comparing on-prem to Azure). Know Reserved Instances, Savings Plans, Spot, and Hybrid Benefit — that's how the exam tests "how do I spend less".

Notes 3.1 Cost management Quiz 3.1 Cost management

Learn

  • Cost factors: resource type, consumption (metered), maintenance state (stopped-deallocated vs stopped), geography, network egress, subscription type
  • Pricing Calculator vs TCO Calculator — planning new spend vs comparing against on-prem
  • Cost Management + Billing — analysis, budgets, cost alerts, exports
  • Tags for cost allocation (department, owner, environment)
  • Savings: Reserved Instances (1 or 3 yr, up to 72% off), Savings Plans (flexible commit), Azure Hybrid Benefit (bring your own Windows / SQL licences), Spot VMs (deep discount, can be evicted)

Study

  • MS Learn · Describe cost management in Azure
  • Azure Pricing Calculator
  • Azure TCO Calculator
  • Cost Management + Billing overview

Hands-on

In the Pricing Calculator, size a small three-tier web app: two B2ms VMs, one Standard SQL DB, 100 GB blob storage, one Application Gateway. Note the monthly total. Change the region to a cheaper one and observe the delta. Add a 3-year Reserved Instance to the VMs and observe the further savings.

In the portal, go to Cost Management and set a $5 budget with an email alert on your subscription — this protects you from bill shock while you keep practising.

Self-check

  • A VM is stopped vs stopped-deallocated — which one is still billing you for compute?
  • You have predictable steady workloads for the next three years. Which savings option?
  • Pricing Calculator or TCO Calculator: you want to convince your CFO that moving your datacentre to Azure will be cheaper over five years?
  • Which cost factor changes if you move a resource from one region to another with everything else identical?

Phase III · Governance
2 h reading
45 m lab

Governance, policy, and compliance

Domain 3's trickiest cluster of near-synonyms: Policy vs RBAC, resource locks vs Policy denials, Purview vs Service Trust Portal. A crisp mental model of "what enforces what, at what scope" is the fastest way to pick these questions apart.

Notes 3.2 Governance & compliance Quiz 3.2 Governance & compliance

Learn

  • Azure Policy — evaluates resources against rules (definitions) and enforces compliance. Answers "what can be done?"
  • Role-Based Access Control (RBAC) — grants identities permissions to perform actions. Answers "who can do what?"
  • Resource locks — Read-Only (no modify or delete) and Delete / CanNotDelete (modify allowed, delete blocked). Applied at subscription, RG, or resource scope
  • Microsoft Purview — data governance, discovery, and compliance across estates
  • Service Trust Portal — Microsoft's public repository of audit reports, ISO / SOC / PCI attestations, and compliance guides
  • Common frameworks: ISO 27001, HIPAA, PCI-DSS, GDPR, FedRAMP

Study

  • MS Learn · Describe features and tools in Azure for governance and compliance
  • Azure Policy overview
  • Azure RBAC overview
  • Service Trust Portal
  • Microsoft Purview

Hands-on

Assign a built-in Azure Policy (for example, "Allowed locations") to your resource group, restricting resource creation to one region. Then try to create a resource in a different region — observe the policy denial.

Apply a ReadOnly lock to your resource group. Try to edit a resource. Remove the lock when done — locks persist across sessions and can block your future labs.

Self-check

  • You want to prevent anyone in your subscription from deploying storage accounts without HTTPS-only. Policy or RBAC?
  • You want to prevent a specific team from deleting a specific database. Lock, RBAC, or both?
  • An auditor asks for Microsoft's SOC 2 Type II report. Where do you find it?
  • Purview or Policy: automatically classify sensitive customer data across storage accounts?

Phase III · Tools & Monitoring

Deployment tools and monitoring — Portal to Advisor

The exam wants you fluent across every way to touch Azure (Portal, CLI, PowerShell, Cloud Shell, ARM/Bicep, Arc) and every way to observe it (Monitor, Log Analytics, App Insights, Service Health, Advisor). Know the five Advisor pillars — reliability, security, performance, cost, operational excellence.

Notes 3.3 Deployment tools Notes 3.4 Monitoring tools Quiz 3.3 Deployment tools Quiz 3.4 Monitoring tools

Learn

  • Interfaces: Azure Portal (GUI), Azure CLI (Bash/cross-platform), Azure PowerShell (Az module), Cloud Shell (browser terminal), Azure mobile app
  • Infrastructure as code: ARM templates (JSON), Bicep (a DSL that transpiles to ARM)
  • Azure Arc — extends Azure management to on-prem, other clouds, and edge
  • Monitoring: Azure Monitor (metrics + logs + alerts), Log Analytics (KQL against logs), Application Insights (APM for apps), Service Health (Azure incidents), Resource Health (your specific resource's status)
  • Azure Advisor — personalised recommendations across five pillars: reliability, security, performance, cost, operational excellence

Study

  • MS Learn · Describe features and tools to manage and deploy Azure resources
  • MS Learn · Describe monitoring tools in Azure
  • Azure Monitor overview
  • Azure Advisor overview
  • Bicep overview

Hands-on

Open Cloud Shell and create a storage account with a single CLI command: az storage account create --name aztest$RANDOM --resource-group <your-rg> --sku Standard_LRS. Compare the experience with clicking through the same in the portal.

Open Azure Advisor and read every recommendation on your subscription. Note which of the five pillars each falls under. Open Service Health to see any current Azure-side incidents.

Self-check

  • Which service tells you about outages Microsoft is having (not your resource being down)?
  • ARM template or Bicep: which is JSON, which is a friendlier DSL that produces the same ARM output?
  • You have a VM running in AWS you'd like to manage centrally alongside your Azure fleet. Which service?
  • Recite the five Azure Advisor pillars from memory.

The final polish, then sit the exam

Do not cram today. Do not take a fresh practice test in the two hours before the exam. Warm up on the cheat sheet, breathe, and go in confident. During the exam, mark-and-move on anything you're unsure about — you have time to return to it.

Cheat sheet One-page warm-up · nothing else

Learn

  • Morning: read the cheat sheet at the bottom of this roadmap. Skim your notes of wrong answers from Days 12–13. Nothing new
  • One hour before: eat light, water, no caffeine spikes. Prepare your testing area (online) or leave for the test centre with 30 minutes to spare
  • During the exam: read every question twice. Watch for the words always, never, only, most, least. Eliminate the obvious wrong answers before choosing between the plausible ones. Flag and return to hard questions
  • Case-study sections, if present, cannot be revisited once submitted. Read the case fully before answering any question in it

Study

  • Official exam page · AZ-900
  • Online exam requirements & system check
  • Pearson VUE test centres

After the exam

You receive your score immediately. If you passed — and you will — your digital badge and certificate are available in your Microsoft Learn profile within 24 hours. Add it to LinkedIn today; the visibility compounds.

If you're going further: your next natural steps are AZ-104 (Administrator), AZ-204 (Developer), AI-900, or SC-900 (Security). Rest for a day first.

Self-check

  • Did you flag and return to at least one question you weren't 100% sure about?
  • Did you finish with time to spare? Ten minutes is comfortable; two minutes is tight.
  • You passed. What's next on your certification path?

The night-before reference.

Every fact below has appeared in the AZ-900 exam bank in one form or another. Rebuild the table from memory on Day 13 and cross-check. If you can produce every row by exam morning, you're operating at the top of the score band.

ConceptThe fact
Passing score700 / 1000. Score reported by domain, not question count.
Regions worldwide60+ Azure regions, more than any other cloud provider. Each is a set of datacentres within a latency-defined perimeter.
Availability ZonesMinimum three physically separate zones in a zone-enabled region. Each is one or more datacentres with independent power, cooling, network.
Region pairsEvery region paired with another in the same geography (usually 300+ miles apart) for disaster recovery. Updates staggered across the pair.
Resource hierarchyManagement groups > Subscriptions > Resource groups > Resources. A resource lives in exactly one RG and one region.
Free account$200 credit for 30 days, 12 months of popular services free, plus always-free services (functions, blob to a limit, and more).
VM SLA ladderSingle Premium SSD VM: 99.9%. Availability set: 99.95%. Availability zones: 99.99%.
Storage redundancyLRS = 3 copies in one DC. ZRS = 3 zones. GRS = 6 copies across 2 regions. GZRS = ZRS + geo. RA-* variants add read to secondary.
Blob access tiersHot > Cool > Cold > Archive. Storage cost falls, access cost rises. Archive requires rehydration up to 15 hours.
Load-balancing layersLoad Balancer L4 (TCP/UDP). Application Gateway L7 regional. Front Door L7 global. Traffic Manager DNS-based global.
VPN vs ExpressRouteVPN Gateway: encrypted tunnel over public internet. ExpressRoute: private circuit through a partner, higher bandwidth, higher SLA.
Policy vs RBACPolicy governs what can exist / happen. RBAC governs who can act. They're complementary — use both.
Resource locksReadOnly (no modify or delete) and Delete (modify allowed, delete blocked). Inherit down the scope tree.
Zero Trust principlesVerify explicitly · Use least-privilege access · Assume breach.
Defense in depth layersPhysical · Identity & access · Perimeter · Network · Compute · Application · Data.
Defender vs SentinelDefender for Cloud = CSPM + workload protection + Secure Score. Sentinel = cloud-native SIEM + SOAR.
Entra ID nameFormerly Azure Active Directory. Renamed in 2023. Both names appear in older content; exam uses the new one.
Cost savingsReserved Instances (1 or 3 yr, up to 72% off), Savings Plans (flexible commit), Azure Hybrid Benefit (bring your own Windows/SQL licences), Spot VMs (deep discount, evictable).

Migration ladderThe five R's: Rehost · Refactor · Rearchitect · Rebuild · Replace.
SLA compositionComposite SLA is the product of dependent-service SLAs. Two 99.9% services in series ≈ 99.8%.
VM billing statesStopped: still billed for compute. Stopped-deallocated: not billed for compute. Storage always billed.

§ 06 · Playbook

Exam day, hour by hour.

A calm ritual beats last-minute cramming. Follow the timings below to arrive at the desk in the right state — knowledgeable, rested, unhurried.

T – 12 hours

The night before

  • Skim the cheat sheet once. Do not attempt new material
  • Confirm the exam booking, ID requirements, and time zone
  • Online: run the Pearson VUE system check on the exact device you'll use
  • Lay out ID, water, snacks (if allowed at your centre)
  • Sleep at your normal time. Don't sabotage yourself with an early night your body isn't used to
T – 60 minutes

The final hour

  • Light meal, water, small dose of caffeine only if you use it daily
  • Online proctoring: clear the room, phone away, single monitor, disconnect other displays
  • Test centre: leave 30 minutes early. Bring two forms of ID
  • Read the cheat sheet one last time. Set it aside — no phones or notes allowed in the exam
During the exam

Sixty minutes

  • Read each question twice. Note qualifiers: always, never, only, most, least, best
  • Eliminate obviously wrong answers first, then choose between the two that remain
  • Mark for review any question you're less than 90% on; return with fresh eyes
  • Time budget: about 60–90 seconds per question. Case studies read once, thoroughly, then answer
  • Case-study sections cannot be revisited after submission — check work before hitting next
  • Trust your first instinct on multiple-choice unless you find concrete evidence to change it
One thing not to do: take a full-length practice exam on the morning of the exam. A poor score will rattle you; a great score will make you cocky. The goal by exam-morning is calm confidence, not fresh data.
