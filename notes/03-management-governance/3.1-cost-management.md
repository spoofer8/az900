# 3.1 · Describe cost management in Azure

**Domain 3 · Azure management and governance** — 30–35% of exam

Two calculators for planning, one blade for policing, several levers for saving. Know the difference between the calculators and the *five* main ways to spend less.

---

## Factors that affect cost

| Factor                 | How it affects cost                                                                     |
| ---------------------- | --------------------------------------------------------------------------------------- |
| **Resource type**      | Different services have different pricing models (per-hour, per-request, per-GB, per-execution) |
| **Consumption**        | Metered — you pay for what you use (compute-seconds, GB, transactions)                  |
| **Maintenance state**  | Stopped ≠ deallocated: a stopped VM is still billed for compute                        |
| **Geography (region)** | Prices vary by region; some are cheaper than others                                     |
| **Network egress**     | Data *leaving* Azure is billed. Ingress is free                                         |
| **Subscription type**  | Free trial, pay-as-you-go, Enterprise Agreement, CSP, Dev/Test — different rates       |

### The stopped-vs-deallocated trap

| VM state             | OS shut down? | Compute billed? | Storage billed? |
| -------------------- | :-----------: | :-------------: | :-------------: |
| Running              | No            | Yes             | Yes             |
| Stopped (from OS)    | Yes           | **Yes**         | Yes             |
| Stopped-deallocated  | Yes           | No              | Yes             |

> **Exam-hot** — Stopping a VM from *within the OS* does not stop billing. Only **deallocating** it (from the Azure portal / CLI) does.

### VM state → billing

```
                                     Compute        Storage
                                     billed?        billed?
                                     ─────────      ────────
   Running          ┌───────────┐    YES  ● ● ●     YES  ●
                    │  running  │
                    └─────┬─────┘
                          │  shutdown -h now  (from inside OS)
                          ▼
   Stopped (from OS)      YES  ● ● ●          ← TRAP!
                    ┌───────────┐              Guest-OS shutdown does NOT
                    │  stopped  │              stop compute billing.
                    └─────┬─────┘
                          │  Portal / CLI "Stop"  (az vm deallocate)
                          ▼
   Stopped-deallocated    NO                  ← ONLY this stops compute billing
                    ┌───────────┐
                    │deallocated│
                    └─────┬─────┘
                          │  Delete
                          ▼
   Deleted                NO                  NO   Resource gone
```

## Pricing Calculator vs TCO Calculator

Both are free web tools; they do different jobs.

| Tool                     | Job                                                                                    | Answers                                                                 |
| ------------------------ | -------------------------------------------------------------------------------------- | ----------------------------------------------------------------------- |
| **Pricing Calculator**   | Estimate the cost of new Azure workloads before you deploy                             | *"How much will these VMs, DBs, and storage cost me per month?"*        |
| **TCO Calculator**       | Compare the 5-year total cost of ownership of on-prem vs Azure                         | *"Should we migrate our datacentre to Azure? Will it save money?"*      |

- **Pricing Calculator** → for architects, sizing new deployments
- **TCO Calculator** → for CFOs / decision-makers, migration business case

Both live at `azure.microsoft.com/pricing/`.

## Cost Management + Billing

The Azure portal blade for tracking and controlling actual spend.

### What it does

| Feature                  | Purpose                                                                            |
| ------------------------ | ---------------------------------------------------------------------------------- |
| **Cost analysis**        | Break down spend by resource, RG, service, tag, date                              |
| **Budgets**              | Set monthly/quarterly/yearly spend targets, with email alerts at thresholds        |
| **Cost alerts**          | Notifications when actual or forecasted spend crosses a threshold                  |
| **Exports**              | Scheduled CSV export to a storage account for BI / accounting                     |
| **Recommendations**      | Cost-saving suggestions surfaced via Azure Advisor                                 |
| **Invoices**             | View, download, and manage billing invoices                                        |
| **Payment methods**      | Manage credit cards, invoicing, EA agreements                                     |

### Budgets in practice

- Set a monthly spend limit at subscription, RG, or MG scope
- Configure alert thresholds (e.g. 50%, 80%, 100% of budget)
- Attach an **Action Group** to notify people or trigger automated responses
- Budgets do **not** stop resource creation by themselves — pair with automation if you want a hard cap

## Tags

Metadata key-value pairs attached to resources for organisation.

- Up to **50 tags per resource**
- Case-insensitive keys
- **Not inherited** by default from subscriptions or resource groups (an Azure Policy can enforce inheritance)
- Used for: cost allocation, ownership, environment, business unit, project

**Example tag scheme**
```
Environment = Prod | Dev | Test
CostCenter  = CC-1200
Owner       = platform-team@company.com
Application = billing-api
```

Filter Cost Analysis by tag to slice spend by department, product, or environment.

## Ways to save on Azure

### Reserved Instances (RIs)
- Commit to **1 or 3 years** of a specific resource in a specific region
- **Up to 72% savings** vs pay-as-you-go
- Available for VMs, SQL Database, Cosmos DB, Storage, Databricks, and more
- Best for predictable, steady-state workloads

### Azure Savings Plans (for compute)
- Commit to a **fixed hourly spend** ($X / hour) for 1 or 3 years
- More flexible than RIs — applies across VM families, sizes, regions
- **Up to 65% savings**
- Best when you need commitment savings but the exact SKU may change

### Azure Hybrid Benefit
- **Bring Your Own Licence** (BYOL) for existing Windows Server / SQL Server / RHEL / SUSE
- Applies to VMs and PaaS SQL services
- Can be combined with Reserved Instances for **up to 85% off** on some workloads

### Spot VMs
- **Up to 90% discount** on unused Azure capacity
- **Can be evicted** at any time (with 30-second notice) when Azure needs the capacity
- Best for: interruptible batch jobs, dev/test, big-data workloads with checkpointing

### Dev/Test subscriptions
- Discounted rates for non-production workloads
- Requires Visual Studio subscription
- Only for dev/test — using them for prod is a licensing violation

### Consolidated summary

| Option              | Discount       | Commitment    | Flexibility    | Best for                          |
| ------------------- | -------------- | ------------- | -------------- | --------------------------------- |
| Reserved Instances  | Up to 72%      | 1 or 3 yr     | Low            | Steady, predictable workloads     |
| Savings Plan        | Up to 65%      | 1 or 3 yr     | Medium         | Committed spend, changing SKUs    |
| Hybrid Benefit      | Up to 85% combined | Existing licence | High         | Windows / SQL migrations          |
| Spot VMs            | Up to 90%      | None          | Fully flexible | Interruptible workloads           |
| Dev/Test subs       | Reduced rates  | None          | High           | Non-prod                          |

## Cost optimisation habits

- **Right-size** VMs — Advisor recommends this
- **Deallocate** VMs when idle (dev boxes overnight)
- Use **autoscale** so you scale down at low demand
- **Delete unused resources** — orphan disks, old snapshots, empty storage accounts
- **Move data to cheaper tiers** — Cool / Cold / Archive for infrequently accessed blobs
- **Reserve** predictable workloads
- **Tag everything** so you can attribute spend
- Set **budgets and alerts** at RG / subscription level

---

## Gotchas — cost management

- **Ingress is free; egress is paid.** Cross-region egress costs even more.
- **Stopped ≠ deallocated.** The exam loves this.
- **Budgets alert, they don't block.** Automate a cap if you need one.
- **Tags are not inherited by default.** Use policy to enforce.
- **Reservations don't roll over** — buy for the workloads you actually run.
- **TCO Calculator compares to on-prem.** Pricing Calculator prices new Azure workloads.

---

## Anchor facts

- Six cost factors: **type, consumption, maintenance, geography, network, subscription.**
- **Pricing Calculator** = plan new Azure. **TCO Calculator** = compare to on-prem.
- Cost Management + Billing lives in the Azure portal.
- 50 tags per resource, key-value, not inherited by default.
- Five savings levers: **RI, Savings Plan, Hybrid Benefit, Spot, Dev/Test.**
- Deallocate a VM to stop compute billing.
